Start Django project
i 
                        django-admin startproject [project_name]
i
This command will create a folder named [project_name] in your current directory and auto-generate some configuration files that are neccessary for implementing a django project.
This command will create something like this: 
                                
                                    -[project_name]/
                                        manage.py
                                        |-[project_name]/
                                            |-settings.py
                                            |-urls.py
                                            |-wsgi.py
Example
i
Let we are going to start a project named MyProject, so first go to the directory where you would like to store your code/project and then run the following command in the same directory to create a folder named MyProject containing all neccessary configuration files. django-admin startproject MyProject The file structure will be like this: 
                                        
                                            -MyProject/
                                                manage.py
                                                |-MyProject/
                                                    |-settings.py
                                                    |-urls.py
                                                    |-wsgi.py
NOTE: Never misplace the configuration files.
MORE: https://docs.djangoproject.com/en/3.0/intro/tutorial01/#creating-a-project 
                        python manage.py runserver
i
This command is to start Django server at localhost, where you can see your Django website. Always run this command in the directory where manage.py file is contained. Don't worry you don't need to restart your django server everytime when you make a small change in your code. You can make changes in your code while your server is live and just save your changes and reload your page in your browser, just it. It will be explicitly mentioned whenever Django server will need to be restarted.
To verify your Django project works run this command in the directory where manage.py file is contained. You will see the following output on the command line: 
                            
                                $ python manage.py runserver

                                Watching for file changes with StatReloader
                                Performing system checks...
                                
                                System check identified no issues (0 silenced).

                                You have unapplied migrations; your app may not work properly until they are applied.

Ignore the warnings at this time, we will take care of all these later.
Now it means server is running, then visit http://127.0.0.1:8000/ with your broswer to make sure everything gone well. You'll see a "Congratulations!" page, with a rocket taking off. Wow, It worked. Now you are ready to take on next step.
