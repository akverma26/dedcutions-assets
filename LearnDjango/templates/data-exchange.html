Exchange Data between Django and HTML templates
i
Why? Sometimes It is quite neccessary to exchange data between Django and HTML templates, for example consider a scenario that you have programmed some function in python that takes some input and then do some processing and return some output. Now you want to take that input from user through your HTML page and then provide it to your python-function and at last you want to display that output in your HTML page to your user. But it is not possible without exchanging data between Django and your HTML template.
So let's learn how to do that.
From Django to HTML
i
In previous section we used render() method to return our custom HTML template, that takes a dictionary as its optional third argument, that is named as context. This is the argument that is used to pass our data in our HTML template.
Always remember from Django to HTML template, Data is passed as dictionary i.e. context argument expects a dictionary.
Now, in order to use the context data, in HTML template code section is used, that is similiar to Jinja2. A code section in HTML template is surrounded like {% code_section %} and a variable is surrounded like {{ variable }}.
Please note that in template language, keys of the dictionary provided as context argument to the render() method are used as variable.
For more: https://docs.djangoproject.com/en/3.0/ref/templates/language/
Example
i
Let consider an example, we want to pass some bio-data to our HTML template, then our views.py file will look like this: 
                                        
                                            ...
                                            context = {
                                                'name': 'Ankit Kumar',
                                                'dob': '4/04/404',
                                                'languages': ['Hindi', 'English'],
                                                ...
                                            }
                                            ...
                                            return render(request, 'index.html', context)
Now to show this bio-data put the following code in your HTML template: 
                                        
                                            ...
                                            <div>
                                                Name: {{ name }}
                                                Date of Birth: {{ dob }}
                                                Known Languages : {% for language in languages %} {{ language }} {% endfor %}
                                            </div>
                                            ...
Please note that here name, dob, languages are the keys of the context dictionary, so these keys are used as variable in our template code section and variables are surrounded by {{ }} like {{ name }}, {{ dob }} and {{ language }}.
As you can see above languages variable is not surrounded by {{ }}, because the entire code section of for loop is separately defined by {% %}, hence in these brackets variables are used without brackets but note that the local variable language, that is inside the for loop, is surrounded by {{ }}, to get it's value.
